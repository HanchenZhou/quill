A

轻量整合

把 Export 收进右上角 ⋯ 菜单，Outline 保持原按钮。一个 menu 容纳所有低频文件操作（导出/复制路径/在 Finder 中显示…）， 未来加任何"动作"都进这里，不再污染顶栏。

  • + 改动最小，符合习惯（VSCode/Notion 都这套）
  • + 扩展性强：未来打印/分享/复制路径全进这里
  • − Outline 还是显式按钮，并没真正"减负"

B

边缘细线

把大纲变成右边一条细脊柱：每个标题一道短横，长度对应 h1/h2/h3； 当前阅读位置一道亮 accent + 光晕。鼠标移到这条边缘时整张大纲卡片浮出来。 Export 完全离开顶栏，塞进状态栏右下角，跟主题切换并列。

  • + 顶栏完全干净：只剩 ModeSwitcher
  • + 大纲从"开关一个面板"变成"边角余光"
  • + 跟随阅读位置，是marginalia而不是 nav
  • − 工程量最大：需要监听滚动 + 计算当前 heading
  • − 第一次见的人需要 hover 才发现

C

命令面板

加一个 ⌘K 命令面板做所有动作的总入口。Toolbar 只剩 ModeSwitcher 和一颗 "搜索 / 命令" pill。Outline 仍然存在， 但通过 ⌘⇧L 或 ⌘K 中"显示大纲"调出。 额外彩蛋：palette 里还能直接跳到任何 heading。

  • + 顶栏最干净；扩展无上限
  • + 同时解决"跳到 heading"和"搜文件"的入口
  • + 跟其他现代工具一致（Linear/Raycast/...）
  • − 需要新建 command registry 系统
  • − 鼠标流用户每次都要走两步
