# Quill UI Design

> 范围：桌面端 v0 UI 结构与交互。视觉细节（具体色值、字号、间距）落到实现时再调，本文先定骨架。

## 总体形态

**侧栏 + 右侧编辑/预览区**，一次只编辑一个文档（无 Tab）。两个变体：

- **Workspace 模式**：开了文件夹 → 左侧文件树
- **Single-file 模式**：访达双击 / 拖入单个文件 → 侧栏折起来，整窗只有右侧

## 进入方式

| 入口 | 行为 |
|------|------|
| 菜单/快捷键"打开文件夹" | 进 Workspace 模式 |
| 菜单/快捷键"打开文件" | 进 Single-file 模式 |
| Finder 双击 .md | 进 Single-file 模式 |
| 拖 .md 到窗口 | 等同打开文件 |
| 拖文件夹到窗口 | 等同打开文件夹 |

## 组件细节

### 侧栏（Workspace 模式）
- 顶部：当前文件夹名 · `⊟` 折叠按钮 · `⌕` 搜索图标
- 文件树：递归渲染目录与文件
- **md 文件**：正常显示，鼠标 hover 高亮，点击 = 在右侧打开
- **非 md 文件**：dim 色（≈ 50% 透明度），hover 不变化，点击无反应
- 当前打开文件：高亮背景
- 未保存：文件名右侧加 `•`
- 折叠态：完全消失（不留窄条），靠右侧 `⊞` 按钮重新展开

### 右侧顶栏
- 文件名（无文件夹层级，单文件模式下可考虑显示完整路径，待定）
- 三态切换在右上角：`✎` 纯编辑 · `⇄` 分栏 · `👁` 纯预览
- 当前态高亮，点其他态即切换
- **默认打开模式：分栏**（每次打开新文件都回到分栏，不记忆上一次）

### 编辑区
- CodeMirror 6 + `@codemirror/lang-markdown`
- 内嵌主题切换跟随全局
- 未实现：行号、minimap、查找替换、表格快捷（v0 不做）

### 预览区
- Markdown 解析器后选（unified/remark 链优先）
- 滚动同步：v0 不做（架构不挡）
- 主题：跟随全局

### 状态栏
- 一条低调灰底，约 24px 高
- 内容：字数 · 保存状态（`saved` / `saving…` / `unsaved`）
- 暂不放：光标位置、字符数、语法（之后想加再加）

## 主题

**Light + Dark + 跟随系统**，从一开始就设计双套色变量。

- 全部用 CSS 变量 + Tailwind v4 的 `@theme` / `@variant` 机制
- 切换入口：状态栏右侧（或设置面板，v0 用状态栏）
- 三态：`system` / `light` / `dark`，存在 localStorage
- HTML `data-theme` 属性驱动；Electron `nativeTheme` 监听系统切换

## 拖拽

- 拖 .md → 打开为 Single-file
- 拖文件夹 → 打开为 Workspace
- 拖多个 .md → 打开第一个（v0 不做多 tab）
- 非 md 文件 → 接受拖放但不处理（无明显反馈，沉默忽略）

## 窗口
- 沿用 macOS `hiddenInset` 标题栏
- 默认 1200×800，最小 720×480
- 顶部 header 区域 `WebKitAppRegion: drag`
